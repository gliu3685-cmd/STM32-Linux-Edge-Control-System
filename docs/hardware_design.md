# 硬件设计文档

## 基于 STM32 + FreeRTOS + CAN + Embedded Linux 的智能边缘控制终端系统

---

# 1. 硬件设计概述


本项目采用双处理器协同架构：

- STM32作为实时控制节点
- Embedded Linux作为边缘计算节点


系统通过CAN总线实现两个处理器之间的数据通信。


整体硬件架构如下：
          用户端

            |

         网络通信

            |

    -----------------

    ARM Linux主控

    -----------------

            |

         CAN Bus

            |

    -----------------

      STM32控制节点

    -----------------

      |       |       |

   Sensor  Control  Display


---

# 2. 硬件系统组成


整个系统由以下部分组成：


|模块|功能|
|-|-|
|STM32控制板|实时控制与外设驱动|
|Linux开发板|数据处理与网络通信|
|CAN通信模块|处理器间通信|
|传感器模块|采集环境和设备状态|
|执行模块|控制外部设备|
|显示模块|本地状态显示|
|调试模块|程序下载与调试|


---

# 3. STM32控制节点设计


## 3.1 主控芯片选择


选择：

## STM32F407系列开发板


主要参数：

|参数|说明|
|-|-|
|CPU|ARM Cortex-M4|
|主频|168MHz|
|Flash|1MB|
|RAM|192KB|
|浮点单元|支持|
|CAN|支持|
|DMA|支持|
|SPI/I2C/UART|支持|


---

## 3.2 选择原因


STM32F407具有：

- 较强实时处理能力
- 丰富外设资源
- 支持FreeRTOS
- 支持CAN通信


能够满足：

- 多任务调度
- 传感器采集
- 控制输出
- 工业通信


需求。



---

# 4. Linux边缘计算节点设计


## 4.1 开发板选择


推荐：

Orange Pi Zero 3


主要参数：


|参数|说明|
|-|-|
|处理器|ARM Cortex-A|
|系统|Linux|
|网络|WiFi/以太网|
|接口|UART/SPI/I2C|
|存储|TF卡|


---

## 4.2 功能


Linux节点负责：


### 数据处理

接收STM32上传的数据。


### 网络通信

支持：

- MQTT
- TCP/IP


### 数据管理

负责：

- 数据存储
- 状态管理
- 设备管理

## 5.2 硬件组成


STM32端：

使用：

- STM32内部CAN控制器
- TJA1050 CAN收发器


Linux端：

使用：

- USB-CAN适配器


---

## 5.3 选择CAN原因


CAN具有：

- 抗干扰能力强
- 通信可靠
- 多节点支持
- 工业应用广泛


适用于：

- 工业控制
- 汽车电子
- 智能设备
